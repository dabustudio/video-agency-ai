## עץ ניתוב

### ליד חדש נכנס
1. **Project Manager** → לרשום ב-Supabase (טבלת leads)
2. בדוק סגמנט — אם היי-טק / מותג / מסעדה → **התראה לדביר מיד**
3. **Client Relations** → ליצור טיוטת מייל היכרות (Gmail draft, 24 שעות)
4. **Finance** → להכין טיוטת הצעת מחיר (quote-generator skill)

### הצעת מחיר מתבקשת
1. **Finance** → ליצור לפי `pricing_tiers` ב-`business_context.json`
2. מעל 17,000 ₪ → **עצור, שאל את דביר**
3. הנחה מעל 15% → **עצור, שאל את דביר**
4. **Project Manager** → לעדכן סטטוס ב-Supabase ל-"הצעה נשלחה"

### פולואפ — ליד לא מגיב
1. 3 ימים ללא תגובה → **Client Relations** → טיוטת פולואפ ראשון
2. 7 ימים → **Client Relations** → טיוטת פולואפ שני + **Escalate לדביר**
3. לאחר אישור דביר — לסגור ב-Supabase

### פרויקט פעיל — עדכון
1. **Project Manager** → לעדכן סטטוס ב-Supabase
2. עיכוב > 48 שעות → **התראה לדביר**
3. לקוח לא מגיב > 5 ימים → **Client Relations** + **עדכון דביר**
4. פרויקט נסגר → **Finance** → חשבונית → **דביר מאשר ושולח**

### פרויקט נסגר — לאחר מסירה
1. **Finance** → חשבונית סופית
2. **Client Relations** → מייל סיום + בקשת המלצה
3. **Project Manager** → עדכון Supabase → סטטוס "סגור"

### תוכן ללקוח (סקריפט / בריף / הצעה)
1. **Content** → לבצע, לשמור ב-`output/content/`
2. סקריפט לפרויקט מעל 10,000 ₪ → **דביר עובר עליו לפני שנשלח**

### תוכן פנימי לסטודיו (שיווק / סושיאל / יוטיוב)
1. **Marketing** → מחקר מתחרים → רשימת רעיונות → **הצגה לדביר**
2. **דביר מאשר רעיון** → Marketing → כותב סקריפט
3. **דביר מאשר סקריפט** → Marketing → מתזמן יום צילום ב-Calendar
4. **תוכן לפרסום** → **דביר מפרסם תמיד** — Marketing לא מפרסם לבד

### יום צילום — הכנה
1. **Production Coord** → יום לפני: call sheet + תזכורת לדביר
2. 3 ימים לפני: בדיקת ציוד מיוחד
3. **Production Coord** → לרשום ב-Calendar + `output/production/`

### כסף / תזרים
1. **Finance** תמיד
2. חוב / אי-תשלום → **התראה לדביר**

## כללי זהב

1. **אף סוכן לא שולח ללקוח כסף בלי דביר** — לא הצעה, לא חשבונית
2. **ליד מסגמנט היעד = התראה מיידית** — לא לחכות לדוח שבועי
3. **אין תוכן שיווקי פנימי בלי אישור דביר** — Marketing מציע, דביר מחליט
4. **כל תוצר נשמר ב-output/** — שום דבר לא נאבד
5. **שינוי ב-memory/ = אישור דביר** — לא לדרוס בלי אישור
6. **CRM = Supabase** — לא לסמוך על זיכרון session. תמיד לקרוא ולכתוב ישירות
7. **Tone: חברי אבל אסרטיבי** — בכל תקשורת עם לקוחות
