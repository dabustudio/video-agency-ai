## עץ ניתוב — שכבה תפעולית

### ליד חדש נכנס
1. Project Manager → לרשום ב-`output/projects/leads.json`
2. בדוק סגמנט — אם היי-טק/מסעדה/מותג/משרד פרסום → **התראה לדביר מיד**
3. Finance → להכין טיוטת הצעת מחיר (דרך quote-generator)
4. אם נדרש תוכן → Content

### הצעת מחיר מתבקשת
1. Finance → ליצור לפי `pricing_tiers` ב-`business_context.json`
2. מעל 15,000 ₪ → **עצור, שאל את דביר**
3. הנחה מעל 10% → **עצור, שאל את דביר**
4. Project Manager → לעדכן סטטוס ל-"הצעה נשלחה"

### פרויקט פעיל — עדכון
1. Project Manager → לעדכן סטטוס
2. עיכוב > 48 שעות → **התראה לדביר**
3. פרויקט נסגר → Finance → חשבונית → **דביר מאשר ושולח**

### תוכן (סקריפט / בריף / כתוביות)
1. Content → לבצע, לשמור ב-`output/content/`
2. עולה לדביר רק אם ביקש לאשר
3. תוכן לפרסום בשם Moon Studio → **דביר מאשר תמיד**

### שיווק
1. Marketing → טיוטה
2. תוכן לפרסום → **דביר מאשר תמיד**
3. תגובה חיובית ל-outreach → Project Manager + **עדכון לדביר מיד**

### כסף / תזרים
1. Finance תמיד
2. חוב / אי-תשלום → Finance + **התראה לדביר**

## עץ ניתוב — שכבה אסטרטגית

### סריקת טכנולוגיה (כל שבוע)
1. Tech Growth Manager → סורק מקורות (ראה `agents/tech_growth_manager.md`)
2. מסנן לפי 4 שאלות רלוונטיות
3. מייצר `output/growth/tech/weekly_YYYY-WXX.md`
4. מציג לדביר → **דביר מאשר/דוחה כל פריט**
5. פריטים מאושרים → `output/growth/tech/approved_YYYY-MM-DD.md` → **COO**

### סריקת שוק (כל שבועיים)
1. Business Growth Manager → סורק מתחרים, מגמות, הזדמנויות
2. מייצר `output/growth/business/biweekly_YYYY-MM-DD.md`
3. מציג לדביר → **דביר מאשר/דוחה כל הצעה**
4. פריטים מאושרים → `output/growth/business/approved_YYYY-MM-DD.md` → **COO**
5. ליד פוטנציאלי שזוהה → Project Manager + **עדכון לדביר מיד**

### הטמעת שינוי (COO)
1. COO קורא `approved_*.md` (מ-tech או business)
2. מעריך מורכבות: קל / בינוני / מורכב
3. **מורכב** → מציג תכנית ל**דביר לפני ביצוע**
4. **קל / בינוני** → מבצע + commit מתויג `[coo] implement: ...`
5. מעדכן `output/coo/improvement_log.json`

### סקירה שבועית של מערכת (COO)
1. כל יום ראשון → COO עובר על כל הקבצים (ראה `agents/coo.md`)
2. מזהה בעיות / כפילויות / חוסרים
3. מתקן את מה שניתן → commit `[coo] review: ...`
4. מעלה לדביר מה שדורש החלטה

---

### סיכום יומי (Supervisor)
1. Supervisor → סורק את כל `output/` + `git log --since="24h"`
2. מייצר `output/supervisor/daily_YYYY-MM-DD.md`
3. כולל: מה כל סוכן עשה / למה / יתרון + חסרון לכל פעולה
4. מדד יומי: לידים / פרויקטים / הצעות / commits
5. **Supervisor לא מתקן כלום** — רק מדווח

## כללי זהב

1. **אף סוכן לא שולח ללקוח כסף בלי דביר** — לא הצעה, לא חשבונית
2. **ליד מסגמנט היעד = התראה מיידית** — לא לחכות לדוח שבועי
3. **אין החלטות אסטרטגיות בלי דביר** — Growth מציע, דביר מחליט, COO מבצע
4. **כל תוצר נשמר ב-output/** — שום דבר לא נאבד
5. **שינוי ב-memory/ = אישור דביר** — לא לדרוס בלי אישור
6. **COO לא מטמיע בלי אישור** — שינוי מורכב תמיד עובר דרך דביר קודם
