## עץ ניתוב

### ליד חדש נכנס
1. Project Manager → לרשום ב-output/projects/leads.json
2. בדוק סגמנט — אם היי-טק/מסעדה/מותג/משרד פרסום → התראה לדביר מיד
3. Finance → להכין טיוטת הצעת מחיר
4. אם נדרש תוכן → Content

### הצעת מחיר מתבקשת
1. Finance → ליצור לפי pricing_tiers
2. מעל 15,000 ₪ → עצור, שאל את דביר
3. הנחה מעל 10% → עצור, שאל את דביר
4. Project Manager → לעדכן סטטוס ל"הצעה נשלחה"

### פרויקט פעיל — עדכון
1. Project Manager → לעדכן סטטוס
2. עיכוב > 48 שעות → התראה לדביר
3. פרויקט נסגר → Finance → חשבונית

### תוכן (סקריפט / בריף / תמלול)
1. Content → לבצע, לשמור ב-output/content/
2. עולה לדביר רק אם ביקש לאשר

### שיווק
1. Marketing → טיוטה
2. תוכן לפרסום → דביר מאשר תמיד
3. תגובה חיובית → Project Manager מיד

### כסף / תזרים
1. Finance תמיד
2. חוב / אי-תשלום → Finance + התראה לדביר

## דפוסי זמן
- יום ראשון בבוקר: דוח שבועי → Project Manager
- 1 בחודש: דוח תזרים → Finance
- כל 7 ימים: בדיקת חובות → Finance
- ליד חדש: הצעת מחיר תוך 24 שעות → PM + Finance
- פרויקט נסגר: חשבונית + בקשת המלצה → Finance + Marketing

## כללי זהב
1. אף סוכן לא שולח ללקוח כסף בלי דביר
2. ליד מסגמנט היעד = התראה מיידית
3. אין החלטות אסטרטגיות בלי דביר
4. כל תוצר נשמר ב-output/ — שום דבר לא נאבד
